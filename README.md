# Mo.jrk

منصة عربية احترافية لطلب الخدمات الرقمية، مبنية باستخدام **Next.js App Router + TypeScript + TailwindCSS + Supabase**.

## ما الذي يتضمنه المشروع؟

- واجهة عربية RTL بتصميم داكن احترافي
- الصفحة الرئيسية كاملة مع بانرات وخدمات مميزة وFAQ وشهادات
- صفحات الخدمات، تفاصيل الخدمة، الحساب، الطلبات، السياسات والتواصل
- نظام طلب خدمة يعتمد على **إنشاء الطلب من السيرفر** بدل التسعير من الواجهة
- رفع إثبات الدفع والملفات المرجعية إلى **Buckets خاصة**
- نظام Swap مع احتساب رسوم قابل للتعديل من لوحة الإدارة
- لوحة إدارة تتضمن إدارة الخدمات والطلبات والبانرات ووسائل الدفع والعملاء والإعدادات المالية
- مخطط PostgreSQL احترافي
- سياسات Supabase RLS
- SEO أساسي + robots + sitemap

## التشغيل المحلي أو على Replit

1. ارفع المشروع إلى Replit أو فك ضغطه.
2. أنشئ ملف `.env.local` وانسخ القيم من `.env.example`.
3. ثبت الحزم:
   ```bash
   npm install
   ```
4. شغّل المشروع:
   ```bash
   npm run dev
   ```

## إعداد Supabase

1. أنشئ مشروع Supabase.
2. افتح SQL Editor ونفّذ الملف:
   - `supabase/migrations/001_init.sql`
3. أضف القيم التالية في `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (اختياري لكنه مفيد لبعض المهام الإدارية والسigned URLs)
   - `NEXT_PUBLIC_SITE_URL`
4. تأكد من إنشاء الجداول والـ buckets التالية من خلال ملف الـ SQL:
   - `payment-proofs`
   - `order-reference-files`

## أول مستخدم Admin

بعد إنشاء حساب من الواجهة، غيّر دور المستخدم يدويًا في جدول `profiles` إلى:

```sql
update public.profiles
set role = 'admin'
where id = 'USER_UUID';
```

## ملاحظات مهمة

- الواجهة تعرض بيانات تجريبية تلقائيًا إذا لم يتم إعداد Supabase.
- عند إعداد Supabase، يبدأ المشروع بالعمل على البيانات الحقيقية.
- تم جعل رفع الملفات خاصًا وليس عامًا، ويتم عرضها بروابط موقعة للمستخدم أو الإدارة فقط.
- تسعير الطلبات واحتساب رسوم السواب يتمان في السيرفر عند إنشاء الطلب.
